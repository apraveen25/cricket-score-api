You are a Senior .NET Solution Architect and Backend Engineer.

Your task is to design and generate a **production-ready ASP.NET Core Web API** for a **Cricket Scoring Application**, inspired by platforms like CricHeroes.

---

# 🏏 APPLICATION OVERVIEW

We are building a modern Cricket Scoring platform where users can:

* Register & Login
* Create and manage Teams
* Create Matches (local or tournament)
* Perform **ball-by-ball live scoring**
* View real-time scorecards and statistics
* Receive alerts/notifications
* Track player and team performance

The system should support **scalable, real-time, cloud-native architecture** and be deployable to **Microsoft Azure**.

---

## 1. Authentication & Users

* User Registration (Email/Phone)
* Login with JWT Authentication
* Role-based access (Admin, Scorer, Player, Viewer)
* User Profile (stats, matches played, achievements)

---

## 2. Team Management

* Create Team
* Add/Remove Players
* Assign roles (Captain, Wicketkeeper)
* Team statistics

---

## 3. Match Management

* Create Match (T20, ODI, Test, custom)
* Select Teams
* Define Playing XI
* Toss (with decision)
* Match status (Scheduled, Live, Completed)

---

## 4. Live Scoring Engine (MOST IMPORTANT)

Design a **robust ball-by-ball scoring system**:

Each delivery should capture:

* Over, Ball number
* Batsman, Bowler
* Runs scored
* Extras (Wide, No Ball, Bye, Leg Bye)
* Wicket details (type, fielder)
* Strike rotation

Derived calculations:

* Total score
* Run rate
* Required run rate
* Fall of wickets

## 8. Search & Discovery
